

**Here's how the flow works:**

1. **PDF Upload** → User submits a PDF document

2. **PDF → Images** → Convert PDF to page images (using PyPDF2, pdf2image, or Docling)

3. **Layout Detection** → Analyze the image to detect regions:
   - Text blocks
   - Tables
   - Headers/footers
   - Images/diagrams
   - Creates bounding boxes for each region

4. **Reading Order Sort** → Reorder detected regions logically (top-to-bottom, left-to-right) so text flows naturally

5. **Chunk-wise OCR** → For **each chunk/region**:
   - Crop the specific region from the image
   - Send **only that chunk** to olmocr2-7b
   - Collect raw extracted text
   - Move to next chunk
   - **This is the key improvement** — smaller inputs = better accuracy

6. **Aggregate Text** → Combine all chunks into a coherent document text

7. **Schema-Driven Extraction** → Send the full aggregated text + a JSON schema to gemma2-9b:
   ```json
   {
     "schema": {
       "company_name": "string",
       "invoice_date": "date",
       "line_items": ["array of {description, qty, price}"],
       "total": "float"
     }
   }
   ```

8. **Validate & Return** → Gemma2 structures the extracted data into the schema format, you validate it, return to user.

**Why This Beats Single-Shot:**
- ❌ Old: Throw whole page at vision model → hallucinations
- ✅ New: Process regions → OCR is focused → fewer errors → Schema extraction is clean
